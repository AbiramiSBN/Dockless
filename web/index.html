Raspberry Pi / Linux + ADB + scrcpy USB Wi‑Fi

Mirror Android handhelds on Linux — wired or wireless.

A small Bash script built for Android handhelds without display output and lightweight Linux devices (like Raspberry Pi). It finds your devices, optionally sets up ADB-over-Wi‑Fi, and launches scrcpy with controller-friendly defaults.

Get started See commands
No Electron. No daemons. Just scripts. Works on Raspberry Pi OS and any distro with adb + scrcpy.
Quick copy

Install dependencies and run the enhanced script:

Ubuntu / Debian 
sudo apt update
sudo apt install -y scrcpy android-tools-adb
chmod +x ./wwd.sh
./wwd.sh

Features

A simple “connect + display” flow for Android handhelds where HDMI/DP output isn’t an option.

Device picker

Lists all ADB devices and lets you choose USB, Wi‑Fi, or both.

Wireless setup mode

Switches a USB-connected device to TCP/IP 5555, then connects by IP.

Controller-friendly scrcpy

Uses --gamepad=uhid and sensible quality defaults.

Install

You just need scrcpy and ADB.

Ubuntu / Debian
Run 
sudo apt update
sudo apt install -y scrcpy android-tools-adb
Fedora
Run 
sudo dnf install -y scrcpy android-tools
Android (one-time)
  1. Enable Developer options
  2. Enable USB debugging
  3. Plug in and accept the Allow USB debugging prompt

If you already switched ADB to TCP/IP once:

Run

adb devices

FAQ

My device doesn’t show up

Make sure USB debugging is enabled, the cable supports data, and you accepted the “Allow USB debugging” prompt. Then run adb devices.

Wireless mode keeps disconnecting

Wi‑Fi power saving can kill ADB. Keep the phone awake, stay on the same network, and reconnect with adb connect.
